# 성적기반 배정 (現 구현 정리)

`assignTeamsByScore` (convex/innerwars.ts:310-) 의 최종 구현 내용을 정리한 문서입니다.
설계 검토 과정에서 나온 대안들은 맨 아래 "검토했던 대안"에 남겨두었습니다.

## 요약 (쉬운 설명)

내전에서 "성적기반 배정" 버튼을 누르면, 참가자들을 **실력이 비슷하게 A팀/B팀으로
나눠주는 기능**입니다. 실력은 이렇게 계산합니다.

1. **리그에서 얼마나 잘했는지**와 **내전에서 얼마나 잘했는지**를 각각 봅니다.
2. 리그 성적을 **80%**, 내전 성적을 **20%** 비율로 합쳐서 한 사람당 점수를 하나 만듭니다.
   → "요청하신 대로 리그 성적에 훨씬 더 큰 가점을 줍니다."
3. 경기를 별로 안 뛴 사람이 어쩌다 이긴 걸로 순위가 뻥튀기되지 않도록,
   **경기 수가 적으면 점수를 50%(중간) 쪽으로 살짝 당겨서** 보정합니다.
   (예: 1전 1승 = 승률 100%가 아니라 훨씬 낮게 계산됨)
4. **리그도 내전도 한 번도 안 뛴 사람**은 점수를 매길 근거가 없으므로,
   점수와 상관없이 **항상 순위 맨 마지막**에 둡니다.
5. 이렇게 매긴 점수로 순위를 매긴 뒤, 위에서부터 순서대로 **두 팀의 인원수와
   합산 실력이 비슷해지도록** A팀/B팀에 나눠 배정합니다.
6. 배정이 끝나면 내전 상세 페이지 맨 아래에 "성적기반 배정 결과" 표로
   누가 몇 등이었고 리그/내전 배점이 얼마였는지 볼 수 있습니다.

즉: **리그 성적을 훨씬 크게 반영하되, 몇 판 안 뛰고 반짝한 사람이 고평가되지 않게
보정하고, 아예 기록이 없는 사람은 무조건 꼴찌 취급**하는 방식입니다.

## 점수 계산 방식 (기술 상세)

```
leagueRate    = (leagueWins + K × 0.5) / (leagueGames + K)
innerwarRate  = (innerwarWins + K × 0.5) / (innerwarGames + K)
score         = leagueRate × 0.8 + innerwarRate × 0.2
```

- `K = 4` (`SCORE_SMOOTHING_GAMES`): 스무딩 상수. 경기 수가 적을수록 승률이
  0.5에 가깝게 당겨져서, 표본이 적은 선수가 과대/과소평가되는 것을 막습니다.
- `leagueGames` / `leagueWins`: 전체 리그(모든 시즌 통합) 기준 총 경기 수 / 승수
  (`scores` 테이블, leagueId 구분 없이 전체 합산)
- `innerwarGames` / `innerwarWins`: 전체 내전(현재 배정 중인 내전 제외) 기준
  완료된(`status: "done"`) 경기 수 / 승수 (`innerwarMatches` 테이블)
- 가중치 `0.8 : 0.2`는 `SCORE_WEIGHT_LEAGUE` / `SCORE_WEIGHT_INNERWAR` 상수로
  분리되어 있어 나중에 비율만 바꿔서 튜닝 가능합니다.

**전적이 전혀 없는 참가자**(리그 0경기 + 내전 0경기)는 위 공식상 score = 0.5가
나오지만, 이 경우 점수와 무관하게 정렬 시 항상 맨 마지막으로 보냅니다
(`assignHasHistory` 플래그로 구분). 결과 표에는 "(전적없음)"으로 표시됩니다.

## 팀 분배 방식

정렬된 순위를 `i % 2`로 기계적으로 교대 배정하지 않고, **그리디(탐욕) 방식**을
사용합니다.

- 두 팀의 인원 수 차이가 나면, 인원이 적은 팀에 우선 배정 (팀 크기는 항상 최대 1명
  차이로 유지)
- 인원 수가 같으면, **지금까지 합산 점수가 더 낮은 팀**에 배정

순위 1등부터 순서대로 이 규칙을 반복하면 단순 교대 방식보다 두 팀의 총 실력이
더 고르게 맞춰집니다.

## 결과 확인 화면

배정 시점의 순위·리그 배점·내전 배점·종합 점수를 참가자 레코드에 스냅샷으로
저장해두고, 내전 상세 페이지 맨 아래에 "성적기반 배정 결과" 표로 보여줍니다.
이후 리그/내전 결과가 추가로 쌓여도 이 표는 **배정 당시 기준으로 고정**되어
바뀌지 않습니다 (팀이 왜 이렇게 나뉘었는지 나중에도 그대로 확인 가능).

랜덤 배정, 관리자 수동 배정, 팀 배정 초기화를 하면 이 스냅샷은 자동으로
지워져 표가 사라집니다 (성적기반으로 배정했을 때만 의미 있는 정보이므로).

## 검토했던 대안 (참고용)

처음 설계할 때 고려했던 다른 방식들입니다. 현재는 아래 "채택" 표시된 방식으로
구현되어 있습니다.

- **점수 산출**: 승률+스무딩 방식(✅ 채택) vs. 승수에 단순 가중치만 곱하는 방식
  (구현은 쉽지만 표본 적은 선수 과대평가 문제가 남음)
- **가중치 비율**: 6:4 → 7:3 → **8:2**(✅ 채택, 사용자 요청) — 이보다 더 벌리면
  내전 전적이 사실상 무의미해질 수 있어 8:2를 현재 상한으로 보고 있음
- **전적 없는 선수 처리**: 처음엔 "0.5 중간값 취급"으로 제안했으나, 이후 사용자
  요청에 따라 "항상 최하위"(✅ 채택)로 변경
- **팀 분배**: `i % 2` 기계적 교대 vs. 그리디 방식(✅ 채택)
