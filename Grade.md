# 성적기반 배정 개선 의견

`assignTeamsByScore` (convex/innerwars.ts) 를 리그 성적에 가점을 주는 방식으로
바꾸는 것에 대한 검토 의견입니다.

> **적용 완료**: 리그:내전 = **7:3** 가중치로 구현했습니다 (아래 3-1 승률+스무딩 방식,
> 3-3 그리디 팀 분배 포함). 가중치는 `convex/innerwars.ts`의 `SCORE_WEIGHT_LEAGUE` /
> `SCORE_WEIGHT_INNERWAR` 상수, 스무딩 강도는 `SCORE_SMOOTHING_GAMES` 상수로 빼뒀으니
> 나중에 비율을 조정하고 싶으면 이 값만 바꾸면 됩니다.

## 1. 현재 로직의 실제 동작

먼저 확인이 필요한 부분: 현재 `assignTeamsByScore`는 **리그 승수만** 계산에 사용하고
있고, 내전(`innerwarMatches`) 전적은 전혀 반영하지 않습니다.

```ts
const allScores = await ctx.db.query("scores").take(2000); // scores 테이블 = 리그 전적만
const wins = allScores.filter(
  (s) => (s.homeUserId === p.userId && s.homeScore > s.awayScore) ||
         (s.awayUserId === p.userId && s.awayScore > s.homeScore)
).length;
```

`scores` 테이블은 `leagueId`만 가지고 있고(schema.ts:40-46), 내전 결과는 별도의
`innerwarMatches` 테이블(schema.ts:82-93)에 저장됩니다. 지금 "성적기반 배정"은
사실상 **리그 승수 100%**로만 순위를 매기고 있어서, "리그 성적에 가점을 더 주고
싶다"는 요청은 곧 **내전 전적을 처음으로 반영하면서 리그 쪽에 더 큰 비중을 주는
작업**이 됩니다.

## 2. 현재 방식의 한계 (가점 설계 전에 같이 고려하면 좋은 것들)

- **승수(count) 기준, 승률(rate) 아님**: 리그 경기를 많이 뛴 사람이 승률은 낮아도
  승수가 많으면 상위로 배정됩니다. 참가 횟수가 들쭉날쭉한 소모임 리그 특성상 왜곡이
  꽤 클 수 있습니다.
- **표본이 적은 선수 과대/과소평가**: 1전 1승인 선수가 20전 12승인 선수보다
  위로 잡히는 경우가 생깁니다.
- **전적 0인 선수 처리**: 리그·내전 모두 기록이 없는 신규 참가자는 항상 승수 0 →
  항상 최하위로 정렬됩니다. 신규 인원이 몰리면 한쪽 팀에 쏠릴 수 있습니다.
- **팀 분배 방식**: 정렬 후 `i % 2` 로 A/B를 번갈아 배정(코드 320-326행)하는데,
  이는 "1등 A, 2등 B, 3등 A, 4등 B..." 순서라 인원이 짝수일 때는 꽤 준수하지만,
  홀수 인원이거나 상위권 실력 격차가 크면 팀 간 총점 편차가 커질 수 있습니다.

## 3. 제안하는 방향

### 3-1. 점수 산출: 승률 기반 + 리그 가중치

승수 대신 **승률**을 쓰고, 표본이 적을 때 과대평가되지 않도록 스무딩을 넣은 뒤,
리그/내전 승률을 가중 합산하는 방식을 추천합니다.

```
leagueRate    = (leagueWins + K * 0.5) / (leagueGames + K)
innerwarRate  = (innerwarWins + K * 0.5) / (innerwarGames + K)
score         = leagueRate * W_league + innerwarRate * W_innerwar   // W_league + W_innerwar = 1
```

- `K`는 스무딩 상수(예: 3~4경기 분량). 경기 수가 적을수록 0.5(중간값)에 가깝게
  당겨져서 "1전 1승"이 "20전 12승"을 이기는 일을 막아줍니다.
- `W_league`, `W_innerwar`는 요청하신 "리그에 조금 더 가점"을 반영하는 지점입니다.
  '조금 더' 수준이면 **0.6 / 0.4** 또는 **0.65 / 0.35** 정도를 권장합니다.
  (0.8/0.2처럼 극단적으로 벌리면 내전 전적이 사실상 무의미해집니다.)
- 리그/내전 모두 기록이 없는 선수는 0.5(중간값)로 취급되어 항상 최하위로 밀리지
  않습니다.

### 3-2. 대안: 단순 가중치 (구현이 더 쉬움)

통계적으로 엄밀하진 않지만 이해하기 쉬운 방식도 가능합니다.

```
score = leagueWins * W_league + innerwarWins * W_innerwar   // 예: W_league=1.5, W_innerwar=1
```

승률 스무딩 없이 승수에 가중치만 곱하는 방식이라 구현이 간단하지만, 위에서 언급한
"경기 수 적은 선수 과대평가" 문제는 그대로 남습니다. 빠르게 적용하고 나중에
다듬어도 괜찮다면 이 방식으로 먼저 가는 것도 나쁘지 않습니다.

### 3-3. 팀 분배 방식 개선 (선택)

정렬된 순서를 `i % 2`로 기계적으로 나누는 대신, 매 순서마다 **현재 합산 점수가 더
낮은 팀에 배정**하는 그리디 방식으로 바꾸면 실력 격차가 큰 그룹에서도 두 팀의
총점이 더 고르게 맞춰집니다. 구현 난이도는 크게 다르지 않습니다.

## 4. 제 추천

1. 승률(스무딩 포함) 기준으로 바꾸고, 리그:내전 = **6:4** 가중치로 시작
   (원하시면 7:3까지도 무리 없음)
2. 기록이 아예 없는 선수는 0.5(중간값) 취급
3. 팀 분배는 `i % 2` 대신 그리디(현재 합산 점수 낮은 팀 우선) 방식으로 교체
4. 가중치(`W_league`)는 상수로 빼두면 나중에 값만 바꿔서 튜닝 가능

위 방향으로 동의하시면 바로 구현하겠습니다. 가중치 비율(6:4 vs 7:3 등)이나
승수 방식(3-1 vs 3-2) 중 선호가 있으면 알려주세요.
